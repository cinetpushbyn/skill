# boundaries-and-forbidden.md — 边界与禁忌

这个 Skill 的核心定位是"海报方案展开器"，不是生图工具，不是文案工具，不是审核工具。

---

## 这个 Skill 不该做的事

| 禁忌行为 | 说明 |
|----------|------|
| 写完整生图 prompt | 那是下游 Skill 的工作 |
| 写 negative prompt | 不在范围内 |
| 写模型参数（steps、cfg、seed 等）| 不在范围内 |
| 做最终审美审核 | 不在范围内 |
| 写长篇品牌故事或散文式文案 | 超出 brief 边界 |
| 一次给 5 套完全不同方案 | 会让 brief 失焦 |
| 直接生成成品图 | 不在范围内 |
| 把参考图照抄复刻 | 应该是"参考风格"，不是"复制图片" |
| 忽略产品本身去空谈氛围 | 海报是商业产品，要有产品感 |

---

## 禁忌词清单

以下词语倾向于空泛、失控、没有执行价值，输出时尽量避免：

```
高级感拉满
直接封神
氛围感爆棚
非常惊艳
随便都好看
绝美
超有感觉
完全照着这张图来
越高级越好
多来点元素
```

---

## 不该触发这个 Skill 的情况

| 用户说的话 | 原因 |
|------------|------|
| 「帮我看这张图好不好看」 | 这是审美审核，不是方案展开 |
| 「直接给我最终生图 prompt，越详细越好」 | 超出本 Skill 范围，应由下游处理 |
| 「帮我写品牌故事」 | 不是海报 brief |
| 「随便出 10 个不同版本」 | 会让 brief 失焦，不符合定位 |
| 「评价一下我们的竞品海报」 | 竞品分析不是这个 Skill 的范围 |

---

## 越界风险提示

以下场景容易让这个 Skill 越界，需要主动克制：

1. **用户说"越详细越好"** → 不要写成长篇，保持简洁 brief 格式
2. **用户说"多出几套"** → 最多给一套，方向确定后可以微调
3. **用户给了参考图** → 只提炼风格方向，不要照抄图片内容
4. **用户问"这个方案好不好"** → 这是审核，不在范围内，引导用户去下游处理
