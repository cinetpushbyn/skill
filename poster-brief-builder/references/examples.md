# examples.md — Good Case & Bad Case

用于测试和验证这个 Skill 的行为是否正确。

---

## Good Case

### Good Case 1：日期主题展开

**情境**：上一个 Skill 返回"今天是节气：谷雨"，用户说："继续，按这个做成一版海报方案。"

**正确行为**：

输出一版简洁海报 brief：

```json
{
  "poster_direction": "谷雨节气下的东方茶仓产品海报",
  "title": "雨润百谷，茶藏有度",
  "subtitle": "让好茶在恰好的温湿之间静静陈放",
  "visual_keywords": ["晨雾", "新芽", "润泽", "米白", "留白"],
  "selling_points": ["智能控温湿", "隔潮防霉", "隔光抗氧", "天然等离子"],
  "layout_hint": "产品居中，标题靠右，底部一排卖点图标，整体留白干净"
}
```

**关键点**：有节气意象，有产品功能，标题不促销，卖点简短。

---

### Good Case 2：图片参考整理

**情境**：用户上传参考图后说："按照这种感觉，整理成适合我们茶叶柜的海报方向。"

**正确行为**：

把参考图的风格提炼出来，转化成适合当前产品的 brief，不照抄图片：

```json
{
  "poster_direction": "参考高端极简风格的春日茶仓产品海报",
  "title": "春藏一柜，静养好茶",
  "subtitle": "以东方留白呈现茶仓与春日空间的平衡感",
  "visual_keywords": ["极简", "木质", "浅金", "米白", "春日光感"],
  "selling_points": ["恒温恒湿", "静音存储", "隔光养护"],
  "layout_hint": "单品居中，标题靠右竖向排布，底部保留简洁功能信息"
}
```

**关键点**：参考风格，不复制图片内容，产品功能仍然有体现。

---

### Good Case 3：灵感主题选一个展开

**情境**：用户说："刚才那几个灵感里，选'东方茶仓美学'这个，帮我展开一下。"

**正确行为**：

抓住"东方茶仓美学"这个方向，展开一版简洁 brief：

```json
{
  "poster_direction": "东方茶仓美学主题产品海报",
  "title": "以时藏茶，以度护香",
  "subtitle": "在东方空间美学里，好茶值得一个恰当的居所",
  "visual_keywords": ["禅意", "原木", "茶汤色", "留白", "光影"],
  "selling_points": ["智能控湿", "隔光防氧", "静音运行"],
  "layout_hint": "产品侧置偏左，标题居右竖排，整体色调偏暖，留白多"
}
```

**关键点**：只展开一个方向，不发散，不额外给其他版本。

## Bad Case

### Bad Case 1：要求直接生图 prompt

**情境**：用户说："直接给我最终生图 prompt，越详细越好。"

**错误行为**：

```
Prompt: a high-end product poster of a tea storage cabinet, 
ultra-detailed, 4K, cinematic lighting, negative prompt: blurry, 
low quality, steps: 30, cfg: 7.5...
```

**为什么错**：这已经超出 poster-brief-builder 的边界，生图 prompt 应该由下游处理。

**正确处理**：告知用户这个 Skill 只出 brief，生图 prompt 需要继续到下一步。

---

### Bad Case 2：审美审核请求

**情境**：用户说："帮我看这张图好不好看。"

**错误行为**：开始分析图片的审美质量、构图好坏、色调评分。

**为什么错**：这不是海报方案展开需求，这是审核，不应误触发。

**正确处理**：说明这个 Skill 不负责审核，询问用户是否要基于这张图展开一版海报方案。

---

### Bad Case 3：要求多套不同版本

**情境**：用户说："直接出 10 个完全不同版本，越多越好。"

**错误行为**：

```
版本 1：春日极简风
版本 2：国潮风
版本 3：西方极简风
版本 4：科技感
版本 5：...
```

**为什么错**：会让 brief 失焦，不符合"简洁可落地"的定位，下游也很难用。

**正确处理**：出一个最合适的方向，告知用户可以基于这个方向微调，或者选其他风格后继续展开。

---

### Bad Case 4：长篇散文式输出

**情境**：用户说："给我一个谷雨海报方案。"

**错误行为**：

```
谷雨，是春天的最后一个节气。此时细雨绵绵，万物润泽生长。
我们的茶叶柜，就像大自然最好的守护者，用恒定的温度和湿度，
让每一片茶叶都能在最好的环境里静静陈放......（继续写了 5 段）
```

**为什么错**：这是散文，不是 brief，下游无法直接使用。

**正确处理**：用 JSON brief 格式输出，简洁、可执行。
