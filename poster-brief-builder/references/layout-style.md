# layout-style.md — Skill 文档排版规则

这里规定的是 Skill 文档本身的写法风格，不是海报视觉设计规则。

---

## 标题层级

```
# 一级标题     — 文件名或模块名，每个文件只有一个
## 二级标题    — 主要分区（触发条件、执行步骤、字段说明等）
### 三级标题   — 子分区（具体规则、示例分类等）
```

不要用四级标题（`####`），层级超过三级就该拆文件了。

---

## 列表写法

用列表时：
- 每条一行，不超过 30 字
- 不要在列表里嵌套长段落
- 规则类用 `-`，步骤类用数字 `1. 2. 3.`
- 禁忌/不做什么类用 `|` 表格或 `-` 列表均可

---

## 示例写法

代码块 ` ``` ` 包裹示例，注明语言类型：

````
```json
{ ... }
```
````

不要把示例直接写成正文段落。

---

## JSON 示例写法

- 每个字段单独一行
- key 和 value 之间对齐可读
- 数组值竖向展开（超过 2 个元素时）
- 不要压缩成一行

好的：
```json
{
  "title": "雨润百谷，茶藏有度",
  "visual_keywords": [
    "晨雾",
    "新芽",
    "留白"
  ]
}
```

不好的：
```json
{"title":"雨润百谷，茶藏有度","visual_keywords":["晨雾","新芽","留白"]}
```

---

## 什么内容放 SKILL.md

- 这个 Skill 是什么（一句话定位）
- 触发条件（什么情况用它）
- 执行步骤（1、2、3 几步）
- 输出格式（最简版示例）
- 不做什么（边界说明）
- 指向 references/ 的链接

**SKILL.md 不超过 100 行。** 内容多了就拆进 references/。

---

## 什么内容放 references/

- 字段细则（brief-structure.md）
- 标题写法规则（title-rules.md）
- 卖点写法规则（selling-point-rules.md）
- 完整输出格式和示例（output-format.md）
- 边界与禁忌（boundaries-and-forbidden.md）
- Good/Bad Case（examples.md）
- 文档排版规则（layout-style.md，就是这个文件）

---

## 语言风格

- 大白话，不要故作专业
- 直接说规则，不要铺垫
- 举例子，不要只讲道理
- 能用列表就不用段落
- 不要写"我们认为""建议您""请注意"这类客服腔
