# daily-poster-generator

**服务对象**：RACHING 美晶茶叶柜产品海报  
**出图比例**：9:16 竖版 PNG  
**出图方式**：对话直接触发生图

---

## 触发词

| 触发词 | 说明 |
|--------|------|
| `生成今日海报` | 自动识别今日节点，出最多 2 张 |
| `生成节日海报 <主题词>` | 手动指定节日/节气，出 3 张差异版 |
| `生成节日海报`（无主题词） | 自动回退为「生成今日海报」逻辑 |
| `调试今日海报` | 同自动模式，额外返回完整结构化调试信息 |
| `调试节日海报 <主题词>` | 同手动模式，额外返回完整结构化调试信息 |

---

## 执行工作流

### 【自动模式】`生成今日海报`

```
Step 1  读取今日北京时间日期

Step 2  节点识别（只查 A/B/C 三类）
        ├─ A类：二十四节气（静态日期表，见 references/nodes.md）
        ├─ B类：中国传统节日（预计算表，见 references/nodes.md）
        └─ C类：商业节日（公历固定日期，见 references/nodes.md）

        命中优先级：B类 > A类 > C类
        同日多个命中：按优先级取最多 2 个

        ❌ 今日无命中 → 执行【预热策略】：
           向后查找未来 7 天内最近的 A/B/C 节点
           ├─ 找到 → 以「X天后是[节点名]」为主题，进入预热出图
           └─ 7天内也无节点 → 进入【季节品牌海报】分支

Step 3  对每个入选节点，执行【单张出图流程】

Step 4  返回结果
```

### 【手动模式】`生成节日海报 <主题词>`

```
Step 1  解析主题词
        ├─ 主题词为空 → 回退自动模式
        └─ 有主题词 → 智能纠错（见 references/nodes.md §七）

Step 2  验证主题词合法性（必须为节日/节气类）
        └─ 不合法 → 返回错误说明，终止

Step 3  生成 3 套差异方案
        每套在以下维度有实质差异：场景 / 光线 / 构图 / 表达重点

Step 4  对 3 套方案各执行【单张出图流程】

Step 5  返回 3 张结果
```

### 【季节品牌海报】分支（7天内无任何节点时触发）

```
根据当前月份判断季节主题：
├─ 1-2月   → 年初开门红 · 藏茶迎新
├─ 3-4月   → 春茶正当时 · 锁鲜守味
├─ 5-6月   → 夏日保茶
├─ 7-8月   → 防暑护茶
├─ 9-10月  → 秋茶收藏季
└─ 11-12月 → 冬藏，好茶静待时光

主标题：2-4字品牌感词组（不用节日名）
展示全部四项功能点
执行【单张出图流程】
```

---

### 【单张出图流程】

```
Step A  生成营销文案（见 references/prompts.md §一）
        输出两个视角：
        ├─ 视角一：场景共鸣版（适合小红书/朋友圈）
        └─ 视角二：功能直击版（适合干货/卖点解析）
        每个视角包含：标题 / 正文 / Tags

Step B  选取功能点
        ├─ 节日/节气海报：选最相关的 1-2 个功能点
        └─ 季节品牌海报：全部四项功能点
        注：如触发词包含「智能控温湿/隔潮防霉/隔光抗氧/天然等离子」
            则强制展示全部四项功能图标

Step C  生成海报提示词（见 references/prompts.md §二）
        同时输出：
        ├─ 中文提示词（结构化，按固定段落顺序）
        └─ 英文提示词（严格对齐中文结构翻译）
        结尾固定追加参数（不可省略）：
        --ar 9:16 --quality ultra --resolution 2K --style photorealistic
        --lighting global illumination, diffuse soft morning light, no hard shadows, no light beams
        --dof shallow --format poster --no blur --no watermark --no text distortion
        反向提示词：使用 references/prompts.md §三 中的全局模板

Step D  生成图片（直接对话触发）
        说出：生成图片：[英文正向提示词] --no [反向提示词]
        垫图：使用产品白底图 assets/products/meijing-v1-white-bg.png

Step E  基础质检
        检查：产品主体是否清晰可见 / 占比是否合理 / 位置是否正常
        ├─ 通过 → 保存 PNG + 同名 txt（含中英文提示词）
        └─ 未通过 → 返回图片 + 标记「低质量候选图」，不保存

Step F  返回结果
```

## 预热海报说明

当今日无节点命中、但未来 7 天内有节点时，生成预热海报：

- 主标题：节点名（如：谷雨）
- 副标题：预热感文案（如：`X天后，为春茶备好守护`）
- 功能点：按节点匹配选 1-2 个
- 标记为预热图，文件名加 `_预热` 后缀

---

## 文案生成规则

### 标题字数
- 节日/节气/预热海报：主标题 = 节点名（2-4字）
- 季节品牌海报：主标题 = 品牌感短句（2-4字）
- 副标题：6-10字，不写解释性长文案

### 功能点匹配

| 功能点 | 优先匹配节点/场景 |
|--------|-----------------|
| 智能控温湿 | 所有节气、梅雨季、回南天、冬藏 |
| 隔潮防霉 | 谷雨、清明、梅雨季、回南天 |
| 隔光抗氧 | 夏至、大暑、日常品牌 |
| 天然等离子 | 中秋、年货、礼赠、季节品牌 |

---

## 保存规则

| 条件 | PNG | TXT |
|------|-----|-----|
| 质检通过 | ✅ `output/年/月/日期_主题_模式_序号.png` | ✅ 同名 txt，含中英文提示词 |
| 低质量候选图 | ❌ 不保存 | ❌ 不保存 |

## 错误处理

| 情况 | 处理 |
|------|------|
| 手动模式主题词不合法 | 返回错误说明 + 合法示例，终止 |
| 生图失败 | 直接报错，不重试 |
| 质检未通过 | 返回图片 + 低质量标记，不保存 |

---

## 禁忌

- 不允许在节气/节日海报展示全部四项功能点（仅在季节品牌海报/含功能触发词时展示全部）
- 不允许将低质量候选图写入 output 目录
- 不允许主题词为产品词、功能词、品牌词
- 不允许副标题超过 10 字或写成解释性长句
- 不允许生图失败时静默跳过

## Good Case

**Case 1：今日有节点（谷雨）**
- 命中：谷雨(A类)
- 主标题：谷雨 / 副标题：一柜锁住春茶香
- 功能点：智能控温湿
- 营销文案：两视角（小红书风格 + 功能直击）
- 中英双语提示词各一份

**Case 2：今日无节点，未来 7 天有节气（如 3 天后谷雨）**
- 主标题：谷雨 / 副标题：3天后，为春茶备好守护
- 标记为预热图，文件名含 `_预热`

**Case 3：7天内均无节点 → 季节品牌海报（4月）**
- 主标题：锁鲜 / 副标题：春茶正当时，一柜守味
- 展示全部四项功能点

**Case 4：手动模式 中秋（3 张差异版）**
- 版本A：室内落地窗，暖金月光，居中构图，仪式感礼赠
- 版本B：半开放庭院，冷蓝月光+烛光，三分法，自然意境
- 版本C：室内书房，台灯暖光，俯视角，日常储茶×节日礼遇

---

## Bad Case

**Bad Case 1**：节气海报展示全部四项功能点 → 视觉过载，除非触发词含功能关键词

**Bad Case 2**：今日无节点时直接出「日常品牌海报」，不查未来 7 天 → 漏掉预热机会

**Bad Case 3**：`生成节日海报 智能控温` → 应报错「主题词不合法」

**Bad Case 4**：低质量候选图被写入 output/ → 严禁
