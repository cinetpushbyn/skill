# nodes.md — 节点清单与判断规则

> 本文件定义所有受支持的节点类型、完整清单、日期判断逻辑、优先级规则与出图数量规则。

---

## 一、节点分类体系

```
节点
├── A类：二十四节气（按农历/天文历计算）
├── B类：中国传统节日（农历固定日期）
├── C类：常见商业节日（公历固定日期）
└── D类：固定营销节点（产品定制节点，第一版）
```

优先级：B类 > A类 > C类 > D类（用于多节点命中时的评分基准，见评分规则）

## 五、D 类：固定营销节点（第一版）

> 第一版选取 6 个高频节点，后续可按需扩展。

| 节点名称 | 触发日期区间 | 判断逻辑 | 核心功能关联 |
|----------|-------------|----------|-------------|
| 春茶季 | 3月15日 - 4月30日 | 公历区间匹配 | 隔潮防霉、智能控温湿 |
| 梅雨季 | 6月10日 - 7月10日（长江中下游参考） | 公历区间匹配 | 隔潮防霉、智能控温湿 |
| 冬藏季 | 11月15日 - 12月31日 | 公历区间匹配 | 智能控温湿、隔光抗氧 |
| 回南天 | 2月15日 - 3月31日 | 公历区间匹配 | 隔潮防霉 |
| 中秋送礼季 | 中秋节前 14 天 - 中秋节当天 | 依赖 B 类中秋日期 | 天然等离子、智能控温湿 |
| 年货季 | 腊月十五 - 除夕 | 依赖 B 类农历转换 | 全部四项 |

### D 类判断逻辑
- 区间型：公历区间直接匹配，含首末两日
- 依赖型：先解算对应 B 类节日日期，再计算偏移区间

### D 类注意事项
- D 类与 A/B/C 类同日命中时，评分叠加（不覆盖）
- D 类单独命中时，可选择附带对应功能强化表达

## 六、节点优先级与评分规则

### 6.1 基础分

| 节点类型 | 基础分 |
|----------|--------|
| B 类（传统节日） | 90 |
| A 类（节气） | 80 |
| C 类（商业节日） | 70 |
| D 类（营销节点） | 50 |

### 6.2 加分项

| 加分条件 | 加分值 |
|----------|--------|
| 节点与茶叶/储茶强相关（如谷雨、春茶季、中秋） | +10 |
| 节点社会识别度极高（如春节、中秋） | +5 |
| 多类型节点同日命中（叠加） | +5 / 个 |
| 节点自带视觉氛围感强（如清明、冬至） | +3 |

### 6.3 评分示例

```
2026-04-20（谷雨）
├── A类 谷雨：80 + 茶叶相关+10 + 视觉氛围感+3 = 93
└── D类 春茶季（3/15-4/30 区间内）：50 + 茶叶相关+10 + 多节点叠加+5 = 65

最终选谷雨（93 分），春茶季可作为副主题融入文案，不单独出图
```

### 6.4 出图数量规则

| 情况 | 出图数 | 说明 |
|------|--------|------|
| 自动模式，0 个节点命中 | 1 张 | 日常品牌海报 |
| 自动模式，1 个节点命中 | 1 张 | 该节点唯一方案 |
| 自动模式，2 个节点命中 | 最多 2 张 | 取评分最高的 2 个 |
| 自动模式，3 个及以上节点命中 | 2 张 | 取评分最高的 2 个 |
| 手动模式（任何情况） | 3 张 | 固定 3 张差异版本 |

## 七、智能纠错规则（手动模式）

手动模式输入主题词时，支持以下纠错处理：

| 输入形式 | 纠错结果 |
|----------|----------|
| `谷雨节` | → `谷雨`（去除「节」尾缀） |
| `冬至节` | → `冬至` |
| `中秋节` | → `中秋节`（B 类本身含「节」，保留） |
| `春节` | → `春节` |
| `谷語` | → 尝试模糊匹配「谷雨」，匹配成功提示：「已识别为「谷雨」，继续执行中…」 |
| `七夕` / `乞巧节` / `中国情人节` | → 均识别为「七夕节」 |
| `妇女节` / `女神节` / `女王节` | → 均识别为「妇女节」 |

纠错失败处理：
- 如无法确定对应节点，返回错误说明，列出所有合法节点名，**终止执行**
