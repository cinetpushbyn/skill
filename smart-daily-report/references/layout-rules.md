# 智能复盘日报 · 排版与输出规范

> **重要声明**：本文件是日报输出的唯一排版权威来源。
> SKILL.md 的 Step 5 在组装日报时，必须 100% 照搬本文件中定义的模板格式，
> 不得自行发明或修改任何标点、空格、括号或换行规则。


---

## 一、标准输出模板

Agent 生成日报时，必须严格使用以下模板，逐字对照，不得变形：

```
{TODAY}工作总结（工作汇总及进度）：
1.{工作项1}
2.{工作项2}
...（工作项数量根据实际会话内容确定，通常 2~5 条）

{TOMORROW}工作计划(明日工作安排）：
1.{计划项1}
2.{计划项2}
...（计划项数量与今日工作量相匹配，通常 2~4 条）
```

### 模板字段说明

| 字段占位符 | 来源 | 格式示例 |
|------------|------|----------|
| `{TODAY}` | Step 2 · 物理时间锚定获取的真实今日日期 | `2026年3月30日` |
| `{TOMORROW}` | Step 2 · 物理时间锚定推算的明日日期 | `2026年3月31日` |
| `{工作项N}` | Step 3 · 工作量解构后的专业化工作项 | 见第二节 |
| `{计划项N}` | Step 4 · 常识顺延推演的明日计划项 | 见第二节 |

## 二、工作项与计划项写作规范

### 2.1 工作项书写标准

工作项描述须同时包含**过程信息**与**结果状态**，两者��一不可：

```
[职场动词] + [工作对象] + [过程描述（可选）] + [，] + [结果状态]
```

**示例（合规）**：
```
进行产品包装文案的3轮评估与修改，最终完成8601型号包装定稿
探讨并初步确认DT560C的外观设���方向
完成支付模块核心逻辑开发，通过本地联调测试
```

**示例（违规）**：
```
❌ 搞了下包装文案，弄完了
❌ 随便看了看代码
❌ 大概做了个原型
```

### 2.2 计划项书写标准

计划项描述须体现**具体可执行的下一步动作**，而非模糊意向：

```
[职场动词] + [工作对象] + [交付形式或目标状态]
```

**示例（合规）**：
```
推进8601型号包装的视觉排版与打样对接
输出DT560C外观设计的初版草图或渲染方案
对支付模块进行全流程回归测试并修复潜在异常
```

**示例（违规）**：
```
❌ 我推测明天应该继续弄一下
❌ 明天大概整一整
❌ 可能会做设计
```

## 三、格式细节规范

### 3.1 标点与符号规范

| 规则 | 正确示例 | 错误示例 |
|------|----------|----------|
| 标题行日期与文字之间**无空格** | `2026年3月30日工作总结` | `2026年3月30日 工作总结` |
| 标题括号内文字使用**中文括号** | `（工作汇总及进度）` | `(工作汇总及进度)` |
| 工作计划标题括号使用**半角括号** | `(明日工作安排）` | `（明日工作安排）` |
| 列表序号后**无空格、无句点** | `1.推进...` | `1. 推进...` 或 `1、推进...` |
| 工作项之间**不加空行** | 条目紧密排列 | 条目之间有空行 |
| 今日总结与明日计划之间**保留一个空行** | （见模板） | 两部分紧连或多个空行 |

### 3.2 字符编码规范

- 所有中文内容使用 **UTF-8** 编码
- 年月日使用**中文全角字符**，不使用 `-` 或 `/` 分隔符
  - ✅ `2026年3月30日`
  - ❌ `2026-03-30` / `2026/03/30`

### 3.3 工作项数量规范

| 会话内容量 | 工作项数量建议 | 计划项数量建议 |
|------------|----------------|----------------|
| 轻量会话（1~2个话题） | 2 条 | 2 条 |
| 标准会话（3~5个话题） | 3~4 条 | 2~3 条 |
| 密集会话（6个话题以上） | 4~5 条 | 3~4 条 |

> **原则**：宁少勿滥。只记录有实质性产出的工作，过滤闲聊、重复询问等无效信息。

## 四、合规性自检清单（Step 5 强制执行）

在输出日报前，逐条确认以下项目均已满足：

```
输出纯净性检查：
[ ] 日报第一个字符即为今日日期，无任何前缀文本
[ ] 日报最后一个字符即为最后一条计划项末尾，无任何后缀文本
[ ] 不含"你好"、"好的"、"以下是"等引导性语句
[ ] 不含"以上是您的报告"、"祝工作顺利"、"就这些"等结尾废话

内容准确性检查：
[ ] 日期字段来自真实命令执行或系统上下文，非模型编造
[ ] 明日日期 = 今日日期 + 1天（跨月、跨年时自动正确计算）
[ ] 所有工作项均有真实的会话依据，无凭空添加的内容

格式合规性检查：
[ ] 标题行格式完全符合模板（括号、冒号位置正确）
[ ] 序号格式为 `1.`，无空格无句点
[ ] 今日总结与明日计划之间有且仅有一个空行
[ ] 禁忌词表中的词汇在正文中均未出现
```

## 五、完整输出示例（Golden Sample）

以下为符合所有规范的标准输出样本，可作为格式对照参考：

```
2026年3月30日工作总结（工作汇总及进度）：
1.进行产品包装文案的3轮评估与修改，最终完成8601型号包装定稿
2.探讨并初步确认DT560C的外观设计方向

2026年3月31日工作计划(明日工作安排）：
1.推进8601型号包装的视觉排版与打样对接
2.输出DT560C外观设计的初版草图或渲染方案
```

> **注意**：此样本的标点、空行、括号形式均为规范，任何偏差均视为格式不合规。
