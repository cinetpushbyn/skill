# 案例参考

## Good Cases（正确触发示例）

---

### Good Case 1：日期触发

**用户说：** "今天自动看看是不是节气，给我一个适合做茶叶柜海报的主题。"

**判断：** 明确提到"今天""节气"，属于 `date` 触发。

**正确输出：**

```json
{
  "trigger_type": "date",
  "is_special_day": true,
  "day_type": "solar_term",
  "theme": "谷雨"
}
```

说明：今天是谷雨，适合作为茶叶、春耕、养生类产品海报的主题起点。

---

### Good Case 2：图片参考触发

**用户操作：** 上传一张产品海报图，说"我想做一张这样的样式、这样的主题照片。"

**判断：** 有图片 + 有"想要类似样式"的意图，属于 `image_reference` 触发。

**正确输出：**

```json
{
  "trigger_type": "image_reference",
  "reference_summary": {
    "style": "高端极简产品海报",
    "theme": "春日东方茶仓",
    "composition": "单品居中，右侧竖排标题，底部卖点图标",
    "tone": "米白、浅金、干净、留白"
  }
}
```

说明：已提取图片参考摘要，可交由后续 Skill 生成完整 prompt。

---

### Good Case 3：灵感建议触发

**用户说：** "给我几个今天可以做的主题灵感，偏东方一点。"

**判断：** 用户在要创意方向，没有具体日期意图和图片，属于 `inspiration` 触发。

**正确输出：**

```json
{
  "trigger_type": "inspiration",
  "theme_ideas": [
    "春茶焕新",
    "雨润万物",
    "东方茶仓美学",
    "静养陈化",
    "高端家居茶空间"
  ]
}
```

## Bad Cases（不应触发或触发后越界的示例）

---

### Bad Case 1：请求超出本 Skill 边界

**用户说：** "帮我直接写一张完整海报文案，再顺便出图。"

**错误做法：** 直接开始写标题、副标题、卖点文案、生图 prompt。

**正确做法：** 本 Skill 只识别入口，不承担完整海报生成任务。应提示用户这个请求需要进入策划 Skill 和生图 Skill，本 Skill 只能提供基础主题起点。

---

### Bad Case 2：非海报主题触发场景被误触发

**用户说：** "帮我看看这张图怎么样。"

**错误做法：** 因为用户提到"图"就触发本 Skill，开始分析图片风格。

**正确做法：** 用户只是在请求图片评价，没有"想要类似风格""参考这张图做海报"的意图，不应触发本 Skill。直接回应评价需求即可。

---

### Bad Case 3：系统设计需求被误判为触发场景

**用户说：** "帮我做一套完整海报工作流。"

**错误做法：** 把这个请求理解为海报主题触发，开始执行本 Skill。

**正确做法：** 这是系统设计需求，不是本 Skill 的触发场景。本 Skill 不处理完整工作流设计，应告知用户这超出了本 Skill 的职责范围。

---

### Bad Case 4：普通日臆造节日

**实际日期：** 某个普通工作日，既不是节气也不是节日。

**错误做法：** 强行说"今天接近劳动节，可以做劳动主题海报"或编造"今天是茶叶文化日"。

**正确做法：**

```json
{
  "trigger_type": "date",
  "is_special_day": false,
  "day_type": "ordinary",
  "theme": "普通日，交由后续 Skill 处理"
}
```

---

### Bad Case 5：图片参考误判成直接生成

**用户上传图片，说：** "参考这张图做一张产品海报。"

**错误做法：** 直接输出完整生图 prompt，如"a minimalist product poster with centered tea jar, white background, gold typography..."

**正确做法：** 只做参考摘要，输出 `image_reference` 类型的 JSON，交由后续 Skill 处理完整 prompt 生成。
