# 触发规则

## 三类触发条件

---

### 类型 1：date（日期触发）

**触发标志：** 用户请求中明确提到以下任一意图：

- "今天是什么节气"
- "看看今天适不适合做节气主题"
- "自动判断今天日期"
- "今天是节日吗"
- "根据今天的日期给主题"

**判断逻辑：**

1. 获取当前系统日期
2. 对照二十四节气表，判断今天是否为节气
3. 对照常用节日表（春节、元宵、清明、端午、七夕、中秋、重阳、冬至、圣诞等），判断今天是否为节日
4. 以上都不符合，则为普通日

**边界：**
- 不要在普通日臆造节气或节日
- 不要因为"模糊接近某节日"就强行判断为节日
- 节气优先于普通节日（如某天既是节气又是节日，返回节气）

---

### 类型 2：image_reference（图片参考触发）

**触发标志：** 同时满足两个条件：

- 用户上传了一张图片
- 用户的语言表达了"想要类似的""参考这个风格""做一张这种感觉的"

**关键词示例：**
- "想要这样的样式"
- "按这个风格来"
- "类似这种主题"
- "参考这张图"
- "做一张这种感觉的产品海报"
- "这个构图我很喜欢"

**不触发情况：**
- 用户只是上传图片但没说想参考（比如"帮我看看这张图"）
- 用户上传图片是为了让你审核、评价、改稿，不是为了提取风格参考

**误触发规避：**
- "帮我看看这张图好不好看" → 不触发，这是审核请求
- "这张图哪里有问题" → 不触发，这是修改请求
- "帮我复刻这张图" → 可触发，但只做参考摘要，不生成完整 prompt

---

### 类型 3：inspiration（灵感建议触发）

**触发标志：** 用户明确在要主题建议或创意方向，没有具体的日期意图和图片参考。

**关键词示例：**
- "给点主题灵感"
- "给我几个主题建议"
- "今天做什么主题好"
- "想几个方向"
- "给几个创意"
- "有什么主题可以做"
- "偏东方一点的方向有哪些"

**不触发情况：**
- 用户是在直接要完整海报方案 → 不触发本 Skill
- 用户是在要具体文案而不是方向 → 不触发本 Skill

---

## 三类判断优先级

当请求里同时出现多个信号时，按以下顺序判断：

1. 有图片 + 有参考意图 → `image_reference`
2. 明确提到今天日期/节气/节日 → `date`
3. 要创意方向/主题建议 → `inspiration`

如果实在无法判断，先问用户："你是想根据今天日期、参考一张图，还是要几个创意方向？"
