# 文档排版规则

## 适用范围

这份规则管的是：
- `SKILL.md` 怎么写
- `references/` 各文件怎么写
- Skill 执行时输出内容的格式

---

## 标题层级

- `#` 一级标题：文件名或 Skill 名，每个文件只用一次
- `##` 二级标题：大块内容分区（如"触发条件""执行步骤""输出格式"）
- `###` 三级标题：具体类型或子条目（如"如果是 date""如果是 image_reference"）
- `####` 四级标题：只在内容非常细分时才用，不要滥用

不要用加粗代替标题，不要跳级。

---

## 列表写法

- 用无序列表（`-`）列举同级并列内容
- 用有序列表（`1. 2. 3.`）表示有执行顺序的步骤
- 列表项保持简短，一行说清楚，不要写成段落
- 不要嵌套超过两层

---

## 示例写法

- 示例用代码块包裹（` ``` `）
- JSON 示例统一用 ` ```json ` 标注
- 示例前用一句话说明"这是什么情况"
- 示例后可附一两句说明，不超过两句

---

## JSON 示例规范

- 字段名用英文小写 + 下划线（snake_case）
- 字符串值用英文双引号
- 缩进用 2 个空格
- 不在 JSON 里写注释
- 每个触发类型至少提供一个完整示例

---

## `SKILL.md` 应该写什么

- Skill 的定位（一句话）
- 触发时机（何时用、何时不用）
- 执行步骤（Step 1 / 2 / 3）
- 每步的简要说明 + 指向 references/ 的链接
- 不要把详细规则、全部示例、完整禁忌堆在 `SKILL.md` 里

`SKILL.md` 是导航图，不是说明书全文。

---

## `references/` 应该写什么

| 文件 | 内容 |
|------|------|
| `trigger-rules.md` | 三类触发条件的判断规则、关键词、优先级、误触发规避 |
| `output-format.md` | 三类触发的字段说明、JSON 示例、输出简洁度要求 |
| `boundaries-and-forbidden.md` | 不该做什么、禁忌行为、禁忌词、越界风险 |
| `examples.md` | Good Case + Bad Case，附判断说明 |
| `layout-style.md` | 本文件，文档和输出内容的排版规则 |

---

## 语言风格

- 大白话，可执行，不故作复杂
- 每句话说一件事
- 不用"高级感拉满""非常有氛围""非常适合"这类空话
- 禁止词见 `boundaries-and-forbidden.md`
- 能用短语说清楚的，不用完整句子
- 面向"后续 Skill 可以读懂并继续处理"的目标写，不是写给普通读者看的散文
