# layout-style.md

这里说明 Skill 文档的排版规则，以及输出内容的写法要求。

---

## 文档排版规则

### 标题层级

- `#` 一级标题：文件名或文档主题，每个文件只用一次
- `##` 二级标题：主要分区，比如"规则""示例""字段说明"
- `###` 三级标题：具体细分，比如"Good Case 1""字段：image_prompt"
- 不要用四级及以上标题，层级太深难以维护

### 列表

- 并列项用 `-` 无序列表
- 有顺序的步骤用数字列表
- 列表项保持简短，一行说清一件事
- 不要把一段话拆成无意义的多行列表

### 示例写法

- 示例放在代码块里（用 ``` 包裹）
- 示例前写一句说明这个示例是什么情况
- 示例后可以加一段"为什么这是 Good Case / Bad Case"的说明，但要简短

### JSON 示例写法

- 用 ```json 代码块
- 字段名和内容换行对齐
- 不需要写注释，字段含义在旁边的文字里说明

---

## 内容分配规则

### 放在 SKILL.md 里的内容

- 这个 Skill 是什么（一句话定位）
- 触发条件（什么时候用）
- 不触发的情况（什么时候不用）
- 输入类型（3 类输入的说明）
- 执行步骤（3 步）
- 指向 references/ 各文件的链接

SKILL.md 要保持简洁，读完能在 2 分钟内搞清楚这个 Skill 干什么。不要把所有细节都堆在这里。

### 放在 references/ 里的内容

- 详细规则和标准（prompt-structure.md、negative-prompt-rules.md）
- 字段映射逻辑（brief-to-prompt-rules.md）
- 输出格式和示例（output-format.md）
- 禁忌行为和越界风险（boundaries-and-forbidden.md）
- Good Case / Bad Case（examples.md）
- 文档排版规则（layout-style.md，即本文件）

---

## 语言风格

- 简洁、直接、可执行
- 不要写空话，比如"这个 Skill 旨在帮助用户获得更好的创作体验"
- 不要解释显而易见的事，比如"JSON 是一种数据格式"
- 规则要写成可以判断的陈述，而不是模糊的建议
- 中文写作，专有名词保持英文（比如 prompt、brief、JSON）
