# prompt-structure.md

一个合格的 image_prompt 应该包含以下几个部分，按顺序组织。

---

## 结构组成

### 1. 产品主体

明确写出产品是什么，放在 prompt 最前面。

- 写品牌名 + 产品名称
- 写清楚产品的外观特征（比如"茶叶柜""单门冰箱""护肤品套盒"）
- 不要只写"产品"这种模糊词

示例：`RACHING美晶茶叶柜作为画面主体`

---

### 2. 画面场景

说明产品所在的画面环境，让模型知道背景是什么。

- 场景要和主题一致（比如谷雨节气就用晨雾、嫩绿、新芽）
- 场景不要太具体（比如"某某城市的某某地点"），保持通用
- 场景元素写 2-4 个即可，不要堆太多

示例：`谷雨节气氛围，晨雾、嫩绿、新芽、水光元素融入背景`

---

### 3. 视觉风格

说明整体画面的风格方向。

- 可以用形容词组合，比如"现代极简""东方美学""高端商业""轻奢质感"
- 不要堆超过 3 个大方向词，否则互相打架
- 避免空泛词，比如"高级感"单独出现没有意义，要搭配具体描述

示例：`现代商业视觉，高端东方美学，大面积留白`

---

### 4. 色调

明确说明主色调和辅助色。

- 写具体颜色，比如"米白色""浅金色""莫兰迪绿"
- 写色调感觉，比如"整体干净通透""温润柔和""清冷克制"
- 避免"五彩斑斓"或颜色数量过多

示例：`整体米白与浅金色调，干净通透`

---

### 5. 光线氛围

说明光线方向和质感。

- 写光线类型，比如"柔和自然光""侧逆光""柔散射光"
- 写氛围感，比如"温润""清晨感""静谧"
- 不需要写技术摄影术语，模型识别不了太细的参数

示例：`柔和自然光，晨间氛围`

---

### 6. 构图建议

告诉模型产品怎么放，画面怎么分布。

- 写产品位置，比如"居中单品陈列""左置产品右侧标题"
- 写留白比例，比如"大面积留白""上方三分之一留白"
- 写信息区位置，比如"底部保留功能信息区"

示例：`单品居中陈列，标题信息靠右排布，底部保留简洁功能卖点信息区`

---

### 7. 商业海报感

加上几个词说明这是一张商业海报，不是摄影作品或插画。

- 常用词：`适合品牌广告传播`、`商业海报质感`、`品牌宣传视觉`
- 帮助模型理解输出用途，调整整体气质

示例：`适合品牌广告传播，真实产品质感`

---

### 8. 排版意图（可选）

如果 brief 里有标题或卖点，可以在 prompt 末尾简单说明排版意图。

- 不要写具体文字内容（比如标题的具体词），只写排版逻辑
- 不要写成段落文案

示例：`标题信息靠右排布，底部保留简洁功能卖点信息区`

---

## 顺序建议

```
产品主体 → 画面场景 → 视觉风格 → 色调 → 光线氛围 → 构图建议 → 商业海报感 → 排版意图
```

整段 prompt 写成一段自然流动的中文描述，不需要分行，不需要加标签。
