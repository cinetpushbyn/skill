## Good Case

### Good Case 1：完整 brief 转 prompt

**输入：**

用户已经有完整 brief，内容包括"谷雨主题、主标题《一雨知春深》、副标题《谷雨时节·新茶上市》、视觉关键词（晨雾、嫩绿、水光、极简）、卖点（恒温储茶、保鲜锁味）、版式（居中单品、标题靠右、底部信息区）"，并说："把这版转成最终生图 prompt。"

**预期输出：**

```json
{
  "image_prompt": "高端东方美学产品海报，RACHING美晶茶叶柜作为画面主体，单品居中陈列，谷雨节气氛围，晨雾、嫩绿、新芽、水光元素融入背景，整体米白与浅金色调，大面积留白，干净通透，现代商业视觉，柔和自然光，画面高级克制，标题信息靠右排布，底部保留简洁功能卖点信息区，适合品牌广告传播，真实产品质感，柜体结构清晰，整体精致稳定",
  "negative_prompt": "杂乱背景，廉价电商风，过度装饰，花哨字体，文字乱码，产品变形，柜体比例错误，模糊，噪点，过饱和颜色，强烈塑料感，低清晰度，背景喧宾夺主"
}
```

**为什么这是 Good Case：**
- image_prompt 包含产品主体、场景、风格、色调、构图、排版意图
- 没有直接把标题文案塞进去，而是转化为排版意图
- 卖点没有写成广告词，只体现为"底部功能卖点信息区"
- negative_prompt 覆盖常见风险，简洁有效

### Good Case 2：参考图 + brief 融合

**输入：**

用户说："参考刚才那张图的感觉（极简构图、单品居中、大面积留白、米白色调），但别照抄，帮我整理成适合我们茶叶柜的 prompt。"

**预期输出：**

```json
{
  "image_prompt": "参考高端极简产品海报的构图逻辑，以RACHING美晶茶叶柜为主体，单品居中展示，右侧保留标题排版区域，底部简洁功能信息区，画面采用米白、浅金、木质暖调，整体干净留白，强调东方审美与商业产品海报质感，春日茶仓主题，柔和光线，空间秩序清晰，产品比例准确，适合品牌宣传使用",
  "negative_prompt": "照抄参考图，构图混乱，低端电商风，产品主体不清晰，背景过杂，颜色脏乱，文字乱码，柜体变形，失焦，噪点"
}
```

**为什么这是 Good Case：**
- 融合了参考图的构图逻辑，但不是照抄
- 明确把产品主体替换为用户的茶叶柜
- negative_prompt 里加了"照抄参考图"这一项，防止模型直接复制

### Good Case 3：简略输入补全

**输入：**

用户给了一个简略的信息："谷雨主题，茶叶柜，简约留白。直接帮我补成能生图的一版 prompt。"

**预期输出：**

```json
{
  "image_prompt": "RACHING美晶茶叶柜作为画面主体，谷雨节气氛围，晨雾与嫩绿元素点缀背景，整体米白色调，大面积留白，干净简约，柔和散射光，单品居中陈列，底部保留简洁信息区，商业产品海报质感，产品结构清晰稳定",
  "negative_prompt": "杂乱背景，廉价电商风，过度装饰，文字乱码，产品变形，模糊，噪点，颜色脏乱，背景喧宾夺主"
}
```

**为什么这是 Good Case：**
- 补全了场景描述和色调，但没有重新发明主题
- 补全内容围绕"生图表达"展开，没有越界做策划
- 产品主体清晰，不被场景喧宾夺主

## Bad Case

### Bad Case 1：主题灵感请求

**输入：**

"给我 10 个节气主题灵感。"

**错误做法：**

本 Skill 直接输出 10 个节气主题灵感。

**正确做法：**

说明这属于主题入口阶段，应该由 poster-theme-trigger 负责，本 Skill 不处理主题生成。

---

### Bad Case 2：审核请求

**输入：**

"帮我看最终出图好不好。"

**错误做法：**

本 Skill 对出图进行评价，说"这张图质感不错但颜色有点偏"。

**正确做法：**

说明本 Skill 只负责生成 prompt，不负责对最终出图做审核或评判。

---

### Bad Case 3：完整工作流请求

**输入：**

"直接给我一套完整海报工作流，从主题到出图。"

**错误做法：**

本 Skill 误触发，一口气把主题触发、海报策划、生图 prompt 全做了。

**正确做法：**

说明这需要依次经过 poster-theme-trigger → poster-brief-builder → poster-prompt-builder 三个阶段，不应由本 Skill 单独承担。
